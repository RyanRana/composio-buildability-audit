03

The agent that did it

The point of the exercise was to do the research with an agent, using Composio's own SDK/MCP in the spirit of the role — and to be honest about where a human was still needed.

1

Fan out — 10 category agents in parallel

One Claude agent per category (10 apps each), run concurrently. Each is handed the app list and a strict output schema.

claude-opus-4-8 · sub-agent orchestration
2

Research from live docs — Composio web tools

Each agent uses Composio-exposed tools (COMPOSIO_SEARCH, FIRECRAWL_SCRAPE) to search the web and fetch the app's real developer docs — not answer from memory.

composio + composio_anthropic · fetch, don't recall
3

Forced structured output

Every agent must finish by calling record_finding() — a tool whose schema is the AppFinding contract. Validation forces the shape; no free-text parsing.

auth · self-serve · api_surface · verdict · confidence · evidence[]
4

Adversarial verification

A second, independent agent re-researches a sample and tries to refute each finding, focused on the two fields that decide everything.

4 verifier agents · 21-app sample · try to break it
5

Human in the loop

A person hand-checked the riskiest claims against primary docs and arbitrated every disagreement before it changed a row.

WebFetch spot-checks + judgment calls

Where a human was actually needed

  • Obscure long-tail apps. fanbasis, iPayX, Paygent, Waterfall — easy to hallucinate an API for. The agent flags these low-confidence; a human opened the docs. (Two were pleasant surprises: fanbasis & iPayX have real self-serve APIs.)
  • Self-serve vs. gated judgment on enterprise apps (DealCloud, Gladly, PitchBook, Commerce Cloud) — the true answer often lives behind a sales call, so the human confirms the gate exists from docs even when pricing is hidden.
  • Arbitrating disagreements. When research and verifier split (Otter, iPayX, Fathom), the human read the primary source and applied the fix.

04

Verification — and how accuracy moved

Accuracy is what matters most, so it's measured, not asserted. On a 21-app stratified sample (both key fields must match the verified truth), here's what each loop bought:

AppPass 0 · memory onlyPass 1 · doc-grounded Verified truthResultChecked byEvidence

What a memory-only pass got wrong

  • It invented "no public API" for fanbasis, iPayX, Otter and Fathom — a systematic bias to under-credit newer / obscure apps. All four actually ship APIs.
  • It over-credited Plaid as self-serve-free, missing the production-approval gate that's the whole story.
  • Net: only 52% of rows fully correct. Fetching the docs is not optional.

What the loops caught — and what still defeated us

  • The adversarial verifier overturned 3 first-pass labels (Otter approval→partnership; iPayX & Fathom trial→free) and killed a wrong "5 free audits" figure.
  • Paygent Connect stayed low-confidence: a white-label NMI gateway with no clean first-party docs — the honest answer is "gated, credentials via reseller."
  • Exact paid-tier gating for some notetakers (Grain, Fathom) is undocumented — we assert the gate exists, not its price.
Built by an agent, verified by an adversary, arbitrated by a human — and honest about all three.
